Obtrace Browser SDK — add observability to your web app.

Obtrace is a hosted SaaS (like Datadog or Sentry). This is a client library.
Install it, add one import, done. DO NOT create wrapper files, providers, or hooks.

Install:
  npm install @obtrace/sdk-browser

Setup (one line, zero config):
  import '@obtrace/sdk-browser/auto'

That single import starts everything. No init call, no config file, no provider.
It reads env vars at build time (VITE_, NEXT_PUBLIC_, REACT_APP_ prefixes).

Explicit init (if needed):
  import { initBrowserSDK } from '@obtrace/sdk-browser/browser'
  initBrowserSDK({ apiKey: '...', serviceName: 'core' })

Identity rule:
- serviceName must match the connected app name in the project, or an explicit alias configured for that app.
- Example: project app `core` may accept alias `web`; ingest normalizes `web` -> `core`.
- Arbitrary service names break repo binding, autofix scope, and replay correlation.

What it captures automatically:
- Errors (unhandled exceptions, promise rejections)
- Session replay (DOM recording)
- Web Vitals (LCP, CLS, INP, TTFB)
- Fetch/XHR requests
- User interactions

Env vars:
  OBTRACE_API_KEY         — from app.obtrace.ai
  OBTRACE_SERVICE_NAME    — connected app name or explicit alias (e.g. "core" or "web")

Docs: https://docs.obtrace.ai/docs/sdks/browser
